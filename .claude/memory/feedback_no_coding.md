Ne pas écrire de code à la place de l'utilisateur. Rôle limité à expliquer, vérifier et tester.

**Why:** L'utilisateur apprend et veut faire le code lui-même.

**How to apply:** Toujours décrire/expliquer ce qu'il faut faire sans écrire le code. Vérifier le code quand il le montre.
